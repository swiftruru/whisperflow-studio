# v1.15.8

Stage-message clarity during transcription, plus a refreshed README to reflect the queue-UX rewrites that landed in v1.15.5 – v1.15.7.

## Changed

- **Silero VAD message now distinguishes first-run download vs warm-cache load** ([python/whisperflow/models/manager.py](python/whisperflow/models/manager.py), [python/whisperflow/transcriber.py](python/whisperflow/transcriber.py), [locales/*/events.json](locales/zh-TW/events.json)) — the old single message "載入 Silero VAD 語音偵測模型（首次使用需下載約 10 MB）…" appeared on every transcription and led users to think either (a) the download had never completed, or (b) Silero VAD was somehow the whole transcription engine. Transcriber now checks `is_silero_vad_cached(torch_hub_dir)` and emits one of two messages:
  - Cold path (cache miss): `首次使用，正在下載 Silero VAD（約 10 MB；Whisper 轉錄前的語音片段偵測元件）…` — makes the "this only happens once" scope explicit and clarifies VAD's role.
  - Warm path (cache hit, the common case): `準備語音片段偵測（Silero VAD）…` — short, no mention of download, no misleading implication.

- **Main transcription phase now says "Whisper transcribing · model {{model}}"** ([python/whisperflow/transcriber.py](python/whisperflow/transcriber.py), [locales/*/events.json](locales/zh-TW/events.json)) — previously the VAD-preparation message stuck on screen for the entire multi-minute Whisper pass, which made it look like only VAD was doing work. Transcriber now re-emits `STAGE_TRANSCRIBING` right after `_ensure_silero_vad()` with a new i18n key `transcribingWhisper` that interpolates the active model name. Users now see a clear `Whisper 轉錄中·模型 large-v2` (or their configured model) as the sticky message during the actual transcription phase.

## Docs

- **README refresh** ([README.md](README.md)) — the UX / i18n sections had drifted since v1.15.5's queue refactor. Updates cover: state-aware Batch Progress card (only visible while running), queue panel's header-level Retry Failed / Clear Finished actions, append-mode scanning with file-existence guarding, the Recent-folders dropdown next to Browse, drag-drop toast specificity, and the current i18n footprint (19 namespaces, ~915 keys per locale).
