# v1.15.2

Patch release fixing a minor i18n glitch where the system tray menu kept its boot-time language after switching UI language from the titlebar.

## Fixed

- **Tray menu labels now follow runtime language changes** ([src/main/tray.js](src/main/tray.js), [src/main/locale-ipc.js](src/main/locale-ipc.js), [src/main/main.js](src/main/main.js)) — the tray already exposed a `refresh()` method but nothing called it; `registerLocaleIpcHandlers` now accepts an `onLanguageChanged` hook, and `main.js` wires it to `trayHandle.refresh()` so the right-click menu updates immediately after the user toggles zh-TW ↔ en. No app restart needed.
