# v1.4.4

Patch release fixing Windows Python auto-detection for pyenv-win and
similar `.bat`-shim installs, plus a one-click ffmpeg install flow so
users on a fresh machine don't have to leave the app to get going.

## Fixed

- **Windows pyenv-win users saw "找不到 Python 3" even when
  `where python` resolved correctly.**  v1.4.2 introduced a Python
  self-report resolver that ran `<candidate> -c "import sys;
  print(sys.executable)"` through `cmd.exe` to walk `.bat` shims such
  as pyenv-win's `shims/python.bat`.  That path tripped over
  `cmd.exe /c`'s documented quirks: the `;` in the `-c` body is a
  command separator at the shell layer, and the outer-quote stripping
  rules are not commutative with the inner quoting we needed.
  Real-world pyenv-win installs failed silently and the preflight
  panel reported no interpreter.

  The fix drops `-c` entirely for `.bat` / `.cmd` shim candidates.
  `src/main/path-resolver.js :: askPythonForSelfPath` now writes a
  two-line `import sys; print(sys.executable)` script to a temp file
  and invokes `cmd.exe /d /s /c ""<shim>" "<tempscript>""`.  With no
  semicolons or embedded quotes on the command line, `cmd.exe` has
  nothing left to mangle.  Direct `.exe` candidates still use
  `execFileSync(candidate, ['-c', ...])`, which bypasses the shell
  entirely.

## Added

- **One-click ffmpeg install button in the preflight panel.**  When
  the ffmpeg check fails, the panel now renders an "安裝 ffmpeg"
  button that opens a themed dialog listing the package managers
  available on this machine (Homebrew, Scoop, winget, Chocolatey,
  apt, dnf, pacman).  The dialog streams install output inline so
  the user can watch the download progress without flipping to the
  Console tab, and re-runs preflight automatically on success.

  - Admin-free managers (`brew`, `scoop`, `winget`) get a
    one-click install button that runs through the app.
  - Managers that need elevation (`choco`, `apt`, `dnf`, `pacman`)
    are detected but not auto-run.  Instead the dialog shows a
    copy-to-clipboard command for the user's own admin terminal.
  - If the preferred manager isn't installed (e.g. Scoop on a fresh
    Windows machine), the dialog links to the manager's own install
    docs so the user can bootstrap it in one step.

- `src/main/package-manager.js` — cross-platform detection +
  streaming install helper, used by the new dialog and available for
  future dependency install flows.
- IPC handlers `pm:detect`, `pm:install`, and `shell:open-external`
  wired through `preload/preload.js` as
  `detectPackageManagers`, `installPackage`, and `openExternal`.
- `src/renderer/components/install-ffmpeg-dialog.js` + matching CSS
  for the themed modal.

## Changed

- `src/main/preflight-checker.js :: validateFfmpeg` attaches an
  `install-ffmpeg` action to its error result so the preflight panel
  can render the new button without hardcoding the check key.

## Migration notes

- No user action required.  Reinstall v1.4.4 over the existing
  install; the bundled venv and downloaded models are preserved.
- Windows users who previously hit the pyenv-win detection bug should
  now see their interpreter picked up automatically on first launch.
