# v1.7.11

Patch release that lets you **drag-drop a folder directly onto the "Media Directory not configured" preflight row**, matching the behaviour of the main `.dir-card` below.

## Added

- **Drag-drop on the preflight "Media Directory not configured" row** — the main tab's media-directory card already accepts drag-dropped folders (or files, in which case the containing directory is used), but the preflight banner at the top of the tab forced users to click the "選擇資料夾 / Choose folder" button and walk a folder picker. The preflight row now registers the same `dragover` / `dragenter` / `dragleave` / `drop` handlers — guarded to only attach on rows whose action is `browse-media-root`, so other preflight rows (venv, ffmpeg, settings) remain unaffected — and routes the dropped path through the same `onApplyDirectory` callback the click handler uses. A subtle dashed-yellow outline appears while hovering a folder over the row to signal it's a valid drop target.

## Migration notes

No user action required.
