# v1.8.1

Patch release that fixes two issues from v1.8.0.

## Fixed

- **Model-missing guard was bypassed when Settings tab was never opened** — the guard that prevents transcription from silently downloading the model from HuggingFace was reading the model name from the Settings form DOM via `collectFormValues()`. If the user never clicked the Settings tab, the form was empty, `selectedModel` was `undefined`, and the entire guard was skipped — letting the transcription kick off a 15-minute silent download. Now reads the model name from `config.json` via the existing `readConfig` IPC, which always returns the on-disk config regardless of which tabs have been rendered.
- **Download card history section had tight top spacing** — when no download was active and only the history list was visible, the heavy `border-top` on `.download-history` made the area between the card title and the history content feel cramped. Removed the border and adjusted margins.

## Migration notes

No user action required.
