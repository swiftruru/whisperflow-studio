# v1.9.1

Patch release fixing the release-notes preview inside the in-app update dialog.

## Fixed

- **Update dialog showed raw markdown in the release-notes preview** — the dialog dumped the GitHub release body verbatim as plain text via `textContent`, so users saw the raw `<!-- download-guide -->` HTML comment, markdown table pipes, `<details>` blocks, and heading markers. The release body is now rendered as formatted HTML with headings, bullet lists, bold, and inline code via a small DOM-only markdown renderer (no `innerHTML` of user-controlled content, so no XSS risk), and the auto-prepended download guide is stripped from the body before rendering so only the version's changelog appears in the dialog.

## Migration notes

No user action required.
