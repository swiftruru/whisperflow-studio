# v1.16.0

Windows runtime robustness — `device=auto` now falls back to CPU when CUDA DLLs are missing, and preflight detects the MSVC runtime that PyAV needs.

## Added

- **Preflight check for Microsoft Visual C++ Redistributable (Windows only)** ([src/main/preflight-checker.js](src/main/preflight-checker.js), [src/main/error-catalog.js](src/main/error-catalog.js), [src/renderer/components/preflight-panel.js](src/renderer/components/preflight-panel.js), [locales/*/preflight.json](locales/zh-TW/preflight.json)) — the system-check panel now verifies that `vcruntime140.dll`, `vcruntime140_1.dll`, and `msvcp140.dll` are present in `%SystemRoot%\System32`. When they are not, the check turns red and offers a **Download VC++ Redistributable** button that opens the official `aka.ms/vs/17/release/vc_redist.x64.exe` installer. Surfaces the real root cause of `ImportError: DLL load failed while importing packet` (raised by PyAV, which faster-whisper imports transitively) the moment the app opens, instead of surfacing it later as a model-list crash or a transcription-time error. Non-Windows platforms skip the check entirely.

## Changed

- **`device=auto` now probes CUDA runtime DLLs before selecting CUDA** ([python/whisperflow/models/faster_whisper_backend.py](python/whisperflow/models/faster_whisper_backend.py)) — faster-whisper's own `"auto"` only checks whether ctranslate2 was built with CUDA support, not whether the runtime DLLs are actually installed. On a Windows box without an NVIDIA stack that mis-selection used to hard-fail with `Library cublas64_12.dll is not found or cannot be loaded` instead of falling back. We now `ctypes.CDLL`-probe `cublas64_12.dll` / `cudnn_ops64_9.dll` (Windows) or `libcublas.so.12` (Linux) before selecting a device, and when the probe fails we silently fall back to `cpu` and coerce incompatible compute types (`auto` / `float16` / `int8_float16`) to `int8`, with a single Chinese warning line in the transcription log explaining what happened.

## Tests

- **New unit tests for the device/compute-type resolver** ([python/whisperflow/tests/test_device_probe.py](python/whisperflow/tests/test_device_probe.py)) — six `monkeypatch`-driven cases cover: explicit `cpu`/`cuda` passthrough, `auto` with CUDA available, and three `auto`-without-CUDA variants (`auto`, `float16`, `int8` compute types) to confirm the fallback path picks the right compute type and emits the warning.
