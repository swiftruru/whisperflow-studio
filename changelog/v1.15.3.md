# v1.15.3

Patch release fixing a drag-and-drop bug where dropping a video file from a different folder onto the media-folder card would leave the media-folder path stale, and show a vague "no new files" toast that hid the real reason.

## Fixed

- **Media-folder path now follows dragged-in files** ([src/main/queue-manager.js](src/main/queue-manager.js), [src/renderer/index.js](src/renderer/index.js)) вҖ” when `addFiles()` successfully queues a file, it now updates `state.rootPath` to the file's parent directory, and the renderer calls `applyDirectory()` to sync the Settings form input and preflight. Dropping a video from a new folder no longer leaves the UI pointing at the previous folder.

- **Precise "skipped" toast messages** ([src/renderer/index.js](src/renderer/index.js), [locales/zh-TW/queue.json](locales/zh-TW/queue.json), [locales/en/queue.json](locales/en/queue.json)) вҖ” `addFiles()` now returns a `skipped` breakdown (`alreadyQueued`, `hasSubtitle`, `unsupported`, `notFound`), and the drop handler shows a specific toast when there's a single reason (e.g. "и·ійҒҺпјҡеҗҢиіҮж–ҷеӨҫе…§е·ІеӯҳеңЁеҗҢеҗҚеӯ—е№•жӘ”") instead of the generic "no new files to add" message.
