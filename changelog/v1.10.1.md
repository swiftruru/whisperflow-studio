# v1.10.1

Patch release fixing a regression in v1.10.0 where fresh installs silently missed every new config field.

## Fixed

- **New config keys missing from fresh installs on v1.10.0** — the 12 new `SETTING` keys introduced in v1.10.0 (`task`, `write_srt` / `vtt` / `txt` / `json`, `output_dir`, `max_line_width`, `overwrite_policy`, `beam_size`, `best_of`, `temperature`, `condition_on_previous_text`, `no_speech_threshold`, `logprob_threshold`, `compression_ratio_threshold`) lived in the developer-local `python/config/config.json`, which is gitignored — so the CI build only shipped `config.example.json` as the first-run template, and that file was never updated. Result: on a fresh v1.10.0 install the Settings → Output and Advanced groups rendered without any of the new fields because `normalizeConfig` had no defaults to merge in. `config.example.json` is now updated with all twelve new keys; upgrading users get the missing fields populated automatically the next time the renderer reads config.

## Changed

- **Model-delete button tinted in the theme's warm peach accent** — the Delete button on installed model rows used the neutral `btn-secondary` pill which looked identical to the Download button on uninstalled rows. It now uses a light peach-tinted outline pill so users can distinguish the two at a glance without the button reading as a jarring red danger slab (still pastel, still on-theme, just clearly "different action than Download").

## Migration notes

- **No user action required.** On first launch after updating, `readConfig()` merges the new defaults into your existing `config.json` and writes the result back. Any values you had already set (e.g. model, language, initial_prompt) are preserved.

## Next steps

- Red pencil: a follow-up tightens the release workflow so the translation-field template and the dev config can't drift again — either by committing a frozen defaults file directly into the bundle or by running a build-time check that every key referenced in `config.metadata.json :: fieldGroups` also exists in the shipped template.
