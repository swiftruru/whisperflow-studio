# v1.10.0

Large feature release covering six previously-hidden backend capabilities, a fully redesigned Settings / Models / About experience, a new Transcript Preview, global search / diagnostics tooling, and a broad i18n pass that cleans up every remaining English string in the Console.

## Added

### Surfacing existing backend capabilities

- **Output / translation settings** — `task` (transcribe ↔ translate-to-English), `write_srt`, `write_vtt`, `write_txt`, `write_json`, `output_dir`, and `max_line_width` are now all exposed in the Settings tab. `bridge/run_cli.py` no longer hard-codes `write_txt=True` / `write_json=True`, so the UI checkboxes finally matter.
- **Advanced Whisper decoder parameters** — `beam_size`, `best_of`, `temperature`, `condition_on_previous_text`, `no_speech_threshold`, `logprob_threshold`, and `compression_ratio_threshold` are exposed behind a collapsed-by-default Advanced group so power users can tune Whisper without editing config files.
- **Output overwrite policy** — new `overwrite_policy` setting: `overwrite` (default), `skip` (leave existing subtitle untouched), or `rename-suffix` (write `name.1.srt`, `name.2.srt`, …). Implemented in `transcriber.py`'s `_write_outputs`; applies per output file.
- **Transcript Preview card** — after each job finishes, the Main tab shows a segment-by-segment preview (timecode + text) with search, copy-all, copy-per-segment, and Reveal-in-Folder. Reads `<basename>.json` when present, falls back to SRT. Closes automatically when the next queued file starts.
- **Changelog viewer** — About tab now has a "Version history" card that opens an in-app modal listing every shipped `changelog/v*.md` entry (sorted newest first) and renders each with the shared markdown-to-DOM renderer. `changelog/` is bundled into `extraResources` so the viewer works in packaged builds too.
- **Diagnostics export** — About tab's Credits card has new "Copy diagnostics" and "Save as file…" buttons. Output includes app version, Electron/Node/Chrome versions, OS + CPU + memory, venv state, GPU probe (CUDA/MPS via a 5-second `import torch` subprocess), models directory usage, a redacted copy of `config.json` / `settings.json`, and the last 500 Console log lines. Home-directory paths are rewritten to `~` so usernames don't leak.

### New top-level behaviours

- **Tray icon + global shortcuts** — macOS menubar / Windows system tray icon with menu items for Show / Start transcription / Scan / Stop / Quit. Two `globalShortcut` bindings: `CmdOrCtrl+Alt+T` (show window) and `CmdOrCtrl+Alt+R` (quick-run). Opt-out via `settings.json :: trayEnabled = false`.
- **File associations** — dropping a media file onto the WhisperFlow Studio icon (or right-click → Open With on supported OSes) now adds that file directly to the transcription queue. Windows / Linux argv and macOS `open-file` events both handled, with a single-instance lock so a second double-click forwards the path to the existing window instead of launching a second copy. Supported formats: `mp4` / `mov` / `mkv` / `mp3` / `wav` / `m4a` / `flac`.
- **Customizable keyboard shortcuts** — six core bindings (Run / Scan / Stop / Save settings / Clear console / Search console) can be rebound per device via a new Settings → App → Keyboard shortcuts card. Click a binding, press the new combination, Esc to cancel. Reset per-binding or all. Stored in `localStorage`.
- **Accessibility controls** — Settings → App → Accessibility adds a Font size selector (Small / Normal / Large / Extra large) and a High-contrast checkbox. Applied instantly, persisted in `localStorage`, and baked into `theme-boot.js` so there's no flash before the first paint.
- **Profile management UI** — the profile switcher on the Main tab now supports **+ New profile** (copies current config), ✎ rename, and ✕ delete with a confirmation dialog. Names are validated (no whitespace / path separators, max 40 chars, `default` reserved).
- **HuggingFace cache import** — a "Scan HuggingFace cache" button on the Models tab finds Whisper models already on disk at `~/.cache/huggingface/hub/models--Systran--faster-whisper-*` and one-click imports them into the app-managed models folder via the existing hard-link / copy fast path. No network, no re-download.
- **Models tab storage info** — new storage summary card at the top shows used bytes (with file count), free disk space, and the full models folder path with a one-click "Open in Finder / Explorer" button.

### Dangerous-action guards

- **Stop Batch / Skip Current confirmation dialogs** — both buttons now open a themed confirm dialog before actually terminating the transcription, so an accidental click mid-run doesn't throw away ten minutes of work. Dialog shows the current file name when available, and the Stop button is styled destructive.
- **Completion log lines on skip / stop** — when the Python process actually exits after a Skip Current or Stop Batch, the Console prints a closing-bracket line ("目前檔案已跳過，接續處理下一個" / "批次已停止") so the user has a clear end-of-lifecycle signal instead of the misleading "…" ellipsis the "waiting to exit" message used to leave dangling.

## Changed

### UI redesign

- **Settings tab, fully redesigned** — single-row sticky header with title + subtitle + segmented control + Save button; grouped cards with meaningful titles (General / Model / Transcription / Output / VAD / Advanced) replace the old flat APP_SETTINGS + SETTING dump; two-column field layout (label + description left, input right) at ≥640 px; max-width 880 px centered; internal state fields (`media_file_path`, `media_file_name`, `missing_count`) hidden from the form. Keyboard shortcuts and Accessibility cards live under a second "App" segment.
- **Models tab, fully redesigned** — new sticky header with title + subtitle + Scan-HF-cache + Refresh buttons (both sharing a unified pill style). Prominent Storage card showing used / free / path / open-folder. Model rows restyled into one continuous flat list with row hover, rounded pills for Download/Delete, and a subtle green tint on installed rows.
- **About tab, flatter** — card shadows removed, borders harmonised with Settings / Models; author link buttons sized to content instead of stretched; content capped at 920 px centered.
- **Main tab progress card** — fully i18n: stats line (Pending / Running / Paused / Done / Skipped / Failed), elapsed / ETA labels, file counter, scan summary, and batch summary all localized.
- **Full-window tabs** — Settings / Models / About no longer show the right-side Console pane; it's only visible on the Main tab where transcription log actually streams. Each tab caps its content at a readable max-width instead of stretching to the edge on wide monitors.
- **Flat card style adopted globally** — base `.card` removed its drop shadow and adopted the softer `surface1` border + `radius-md` corners. State-specific shadows (drag-over ring, etc.) still work.
- **Segmented control visual refresh** — Settings tab's Transcription / App switch now reads as an iOS-style pill track with a contrasting "thumb" on the active segment, so it's obvious it's a switcher rather than two floating buttons.

### Internationalization

- **Duplicate JSON key fix** — `locales/{en,zh-TW}/queue.json` had two `"actions"` blocks; the second (with only `showInFolder`) was silently clobbering the first (with `scan` / `run` / `skipCurrent` / `stopBatch` / …), which is why those button labels kept falling back to English. Merged into one block, seven translated labels are now actually reachable.
- **Status badge language switch** — `setStatus()` now takes a state token (`'idle'` / `'running'` / `'paused'` / …) and does the i18n lookup internally, so switching language mid-transcription no longer leaves the badge stuck on "待機", and `app:language-changed` re-renders without restarting the elapsed timer.
- **App-native Console log lines are fully i18n-driven** — scan lifecycle, transcription start, venv bootstrap, package install flow, pause / resume, skip / stop (both request + completion), and Python's "Transcription completed in X.XXs" (rewritten in `python-runner.js` at stream time since Python itself doesn't know the UI locale) all translate live on language switch.
- **Queue panel fully localized** — "Pending X · Running Y · Done Z", "N/M processed", "Scanned X files in Y folders", "Batch elapsed" / "Batch ETA", item Retry / Remove buttons, and the total/visible/filter/search summary line are all wired to i18n keys.
- **Settings field labels** — every visible field now ships with a human-friendly `.label` translation (e.g. `write_srt` → "輸出 SRT") so the form stops exposing raw JSON identifiers. Original key shows in the field's tooltip for power users.
- **Boot message + stepGuide** — reference the actual localized button label (「掃描缺字幕的檔案」) instead of a hardcoded English "Scan" that no longer exists in Chinese mode.
- **Skip / Stop wait messages rephrased** — removed the misleading "…" ellipsis that made users wait for a non-existent follow-up line. Now "已請求停止批次，等程序結束。" pairs with a later "批次已停止。" completion line so the Console reads as a complete story.

## Fixed

- **Elapsed timer reset on language switch** — `renderCurrentStatus()` unconditionally called `startTimer()` which reset `_timerStart` to `Date.now()`, making the on-screen "00:32" jump back to "00:00" every time the user flipped locales. Fixed by only restarting the timer on state transition INTO running; language-only re-renders reuse the existing `_timerStart`.
- **Scan / Run / Skip Current / Stop Batch labels stayed English in Chinese mode** — same root cause as the duplicate `"actions"` key above; resolving the JSON fixed all four button labels at once.
- **Settings page couldn't scroll** — the old flex layout had `#settings-content` with its own `flex: 1; overflow-y: auto`, which stole the remaining height and made the Accessibility / Shortcuts cards below unreachable behind `.tab-pane`'s `overflow: hidden`. Now the whole `#tab-settings` is the single scroll container, toolbar + segments are sticky at top.
- **Models tab storage bar overflow on narrow widths** — used / free / scan-button crammed onto multiple lines because the flex row had no `flex-wrap` and `white-space: nowrap` on individual spans.
- **Settings sticky header "floating" at narrow widths** — the global 880 px cap applied to the sticky header itself, so when the user scrolled the stuck header was a centered rectangle with empty strips on both sides. Excluded from the cap; inner `.settings-header-row` centers its content while the band spans the full window.
- **`--radius-md` CSS variable was referenced but never declared** — silently fell back to `0` on every card that used it. Declared as `10 px` so Models / Settings / About cards get the intended rounded corners consistently.
- **`run_cli.py` forced `write_txt` / `write_json` to `True`** — the two hardcoded lines overrode whatever the user saved in the Output group in Settings, so unchecking "Emit JSON" had no effect. Removed; the UI now controls this end-to-end.

## Migration notes

- **`config.json` schema grows 12 new keys.** The Python dataclass `TranscribeConfig.from_dict` already ignored unknown keys, so this is backwards-compatible in both directions — an old config still loads, and a new config works on older Python code with the new fields simply ignored.
- **User-created profile directories (`python/config/<name>/`) are now gitignored.** Existing ones stay on disk; no data loss, but `git` will no longer track future profiles. If you want to version a specific profile, force-add it with `git add -f`.
- **No manual step required.** First launch after updating applies the new defaults silently and the new settings groups appear in the Settings tab.
