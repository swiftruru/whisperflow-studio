# v1.13.0

A UI-polish release focused on dropdowns and the profile workflow. Every `<select>` in the app is now a themed custom dropdown that matches the cream/amber palette (goodbye to the dark macOS-native menu mid-palette), and profile management moves out of the Main tab into its proper home in Settings → Transcription with a safer, less-pushy workflow.

## Added

- **Themed custom select** ([src/renderer/lib/themed-select.js](src/renderer/lib/themed-select.js)) — new reusable dropdown component used everywhere a native `<select>` used to render. A visually-hidden real `<select>` stays in the DOM to preserve form value semantics, so `collectFormValues()` and dirty tracking keep working with zero refactor. Full keyboard support (Enter / Space / ↑↓ / Home / End / Esc / Tab), outside-click + page-scroll auto-close, and a single-open-at-a-time invariant.
- **Unsaved-changes hint in profile card** — the `Save current as new profile` button is hidden by default and only surfaces when the settings form has unsaved edits, paired with a soft-yellow hint banner. Previously the button sat in a three-button CTA row that looked like it wanted to be clicked even when you had no reason to.
- **Profile switch safety prompt** — changing the active profile while you have unsaved edits now shows a destructive confirm dialog (`Discard and switch` vs `Stay on current profile`). Cancel reverts the dropdown to the previous value so UI state stays truthful.
- **`settings:dirty-changed` custom event** — `markDirty()` / `clearDirty()` dispatch a `CustomEvent` carrying the new state, so sibling components (the profile card, future UI) can react without polling the save button's class.

## Changed

- **Profile UI moved from Main tab → Settings → Transcription** — profiles snapshot `TranscribeConfig`, so they belong alongside the parameters they capture. The Main tab is no longer cluttered with a profile chip row, and a dropdown scales cleanly to many profiles without wrapping. Rename / delete demoted from button-weight CTAs to low-contrast link-style actions tucked under the dropdown.
- **`createProfile` accepts a seed config** — backend `createProfile(configDir, name, seedConfig?)` now takes an optional in-memory config. The renderer sends the current form state (via the new `buildMergedConfigFromForm()` helper) so `Save current as new profile` truly captures unsaved edits. Previously the backend only copied whatever was already persisted, silently dropping the tweaks you were trying to fork.
- **After `Save current as new profile`, the dropdown switches to the new profile** — and the dirty banner clears, matching the mental model of "Save As". The previously active profile's `config.json` is untouched.
- **`.settings-group-card` no longer clips children** — was `overflow: hidden` for rounded-corner containment, but themed select menus are absolutely positioned and need to spill outside. Verified safe (no child actually bleeds past the radius).

## Fixed

- **Run button on empty queue showed a ghost "transcription complete" toast** — clicking `開始轉錄` without scanning first produced a green success toast and a stale batch summary, because `lastFinishedJob` and `stats` still referenced the previous batch. `triggerRun()` now bails early with an info toast (`佇列中沒有可轉錄的檔案，請先點「掃描缺字幕的檔案」`) before ever hitting the main process.
- **Scrolling inside a long themed dropdown menu immediately closed it** — the page-scroll auto-close listener fired during menu-internal scroll too. Now checks the scroll target against the open dropdown's root and ignores scrolls originating inside it.
- **Profile dropdown layout broke** — the meta-action row was sharing a grid-column `.field-row-bool` with the dropdown and got flex-row'd into a narrow sliver. Dropped `.field-row-bool`, and restructured the field control so the dropdown + rename/delete stack vertically with proper padding.

## Migration notes

- No user action required. Existing profiles continue to work; the switcher just lives in a new location. The `config:profiles:create` IPC payload changed from a bare string to `{ name, seed }`, but the main handler still accepts the old string form for any out-of-tree callers.
