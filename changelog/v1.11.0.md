# v1.11.0

A UX-focused release: the transcript preview grows up from a cramped sidebar card into a full-screen modal, the Settings checkboxes get a proper on-theme design, and a handful of long-standing i18n / status-badge papercuts are fixed.

## Added

- **Full-screen transcript preview modal** — the old 340 px inline card (which forced single-character word-wrap on Chinese transcripts) is gone. A new full-viewport modal (`95 vw × 90 vh`, capped at `1400 × 900 px`) opens from:
  - the `View latest transcript` button in the Main action stack,
  - the `[View Preview]` action on the transcription-complete toast,
  - an eye icon on every row of the **Recent transcriptions** history list.
  The modal is keyboard-friendly (`Esc` to close, `/` focuses search) and persists through subsequent batch runs — it no longer auto-closes when the next job starts.
- **Auto-open preference** — new `轉錄完成後自動開啟字幕預覽` / `Auto-open preview after transcription` checkbox in Settings → Application. Defaults to off; power users can flip it on and get the old "modal pops up automatically" behaviour.
- **Transcript existence IPC** — `transcript:exists` checks the `.json` / `.srt` / `.vtt` files next to a media file without parsing them, so the renderer can hide the eye icon for history rows whose subtitle file was manually deleted from disk instead of popping a "file not found" modal on click.
- **WebVTT preview support** — if `.srt` / `.json` aren't on disk but a `.vtt` is, the preview modal now reads from it (the existing SRT parser already accepts the `.` fractional separator that VTT uses).
- **Info log category** — new log severity filter between Debug and Warning, for Python-side informational messages that aren't warnings but shouldn't read as debug noise.

## Changed

- **Settings checkboxes redesigned on-theme** — replaced the stock 15 px native checkbox (which read as a hard OS-yellow square) with a custom 18 px rounded pill: 2 px flamingo-pink border in the unchecked state, solid yellow fill + pink outline + soft pink glow when checked, dark-mauve checkmark drawn with pure CSS borders (no SVG asset). The `Auto-loop` toggle switch is untouched — its scoped rules still win.
- **Transcript modal close button** — the `✕` character sitting in a default browser button is replaced with a proper 32 × 32 icon button (SVG X, hover tint, focus ring) that matches the rest of the app's header buttons.
- **Keyboard shortcut help modal** — every `<kbd>` pill in the help list is now a fixed 96 px column so `⌘ R`, `⌘ ⇧ S`, `?`, `Esc`, etc. all render at the same visual width instead of auto-fitting to content.
- **Status badge survives language switches** — refactored `setStatus()` to take a state token (`'idle'`, `'running'`, …) and resolve the label via i18n internally, so flipping the UI language while a transcription is in flight no longer leaves the badge stuck on `待機` / `Idle`.
- **Elapsed timer survives language switches** — the timer now guards against resetting `_timerStart` when the status block re-renders from `app:language-changed`. Switching zh-TW ↔ en mid-run preserves the elapsed count instead of snapping back to 00:00.
- **Queue item stage messages colour-coded** — failed rows render the stage message in red + bold, skipped rows in peach + bold, done rows in green, so the terminal state is legible at a glance instead of reading as the same grey subtext.
- **History action buttons drawn as SVG** — the 👁 / 📂 emoji in the Recent-transcriptions list are replaced with inline SVG glyphs inside 26 × 26 pill buttons (themed hover states for each), so they render identically on every OS / font stack.
- **Transcript IPC returns structured errors** — `transcript:read` now returns `{ ok, segments?, source?, errorCode?, message? }` instead of throwing a raw `Error invoking remote method 'transcript:read': …` blob into the UI when the file is gone.
- **Python log localization** — Python-side stdout is rewritten through `LOG_REWRITE_PATTERNS` before it reaches the renderer, so common messages like `Transcription complete in 90.21s` now render in the active UI language.
- **Stop Batch / Skip Current confirmation dialogs** — both buttons now prompt before acting, since mis-clicks during a long transcription were expensive.

## Fixed

- **Duplicate "Stopped by user" in queue items** — when a user cancelled a running job, both `job.error` and `job.stageMessage` contained the stop message and both rendered, producing the line twice. The stage row is now suppressed when it equals `job.error`.
- **Duplicate `actions` key in `queue.json`** broke Scan / Run / Skip / Stop translations — collapsed into a single block so the i18n DOM walker picks up every key.
- **Empty-state error handling** — deleting the transcript file while the app is running no longer pops an IPC-raw error modal; the preview shows the friendly `transcript:empty.notFound` message instead and history rows hide the eye icon on next render.

## Migration notes

- No user action required. The auto-open preference is stored per-device under `localStorage['transcript.autoOpenOnComplete']` and defaults to `false`, so existing installs upgrade silently into the new button-driven UX.
