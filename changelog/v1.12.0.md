# v1.12.0

Adds an end-to-end UI test suite. The app itself doesn't change for end users — this release introduces a Playwright-based E2E harness for development, with five smoke / regression specs that exercise the renderer through the real Electron window.

## Added

- **Playwright E2E test harness** (`e2e/`) — five tests that launch the actual Electron app and drive it like a user:
  - `smoke.spec.js` — window opens, Main tab is active, status badge resolves
  - `navigation.spec.js` — clicks each of the four tabs, asserts the active class and pane visibility
  - `i18n.spec.js` — clicks the `中/EN` toggle, asserts tab labels flip between English and 繁體中文
  - `theme.spec.js` — clicks the moon/sun toggle, asserts `<html data-theme>` flips
  - `shortcuts-modal.spec.js` — `?` key opens the modal, `Esc` closes it
- **Per-test isolation fixture** ([e2e/fixtures/electron-app.js](e2e/fixtures/electron-app.js)) — each test runs against a fresh temp `userData` directory seeded with a known `settings.json`, so tests never read or write the developer's real settings / history / localStorage. Cleaned up after the test.
- **Three npm scripts**:
  - `npm run test:e2e` — CLI mode, prints pass/fail
  - `npm run test:e2e:ui` — interactive Playwright UI (best for demos and debugging)
  - `npm run test:e2e:report` — opens the HTML report from the last run, with traces and screenshots
- **`e2e/README.md`** — explains what E2E tests are, how to run them, what each spec covers, and how to extend the suite (mock IPC, codegen, visual regression).

## Changed

- **`src/main/main.js` gains a `WHISPERFLOW_E2E` env switch** — about 20 lines, fully internal. When the flag is set, the main process:
  - Skips the auto-updater so test runs don't get a "new version available" dialog mid-test
  - Skips tray icon creation so the macOS menubar stays clean across many test runs
  - Routes `settings.json` through `WHISPERFLOW_E2E_USERDATA` (the temp dir from the fixture) instead of the project-root portable settings, even in dev mode
  - **Has zero effect when the env is not set** — normal `npm start` / packaged builds behave identically to v1.11.2.

## Migration notes

- No user action required. Nothing visible in the shipped app changes.
- For contributors: after `npm install`, you can run `npm run test:e2e` immediately. No separate `playwright install` browser download is needed because the suite drives Electron itself, not Chromium.
