# v1.7.12

Patch release that catches the **"user forgot to download the model and just hit Run"** footgun and routes them to the Models tab instead of silently kicking off a 20-minute HuggingFace download.

## Added

- **Missing-model guard on the Run button** — if the model you picked in Settings isn't present in `userData/models/`, clicking the Run button used to silently start a faster-whisper / HuggingFace Hub download in the background. The download could be several hundred MB to 1 GB depending on model size, the UI gave you no progress indicator, and the only clue something was happening was a flood of `HTTP Request: ...` lines streaming into the console — which most users reasonably read as "the app is hung".

  The run-readiness check (`ensureRunReady()`) now calls `listModels()` after the preflight check passes and compares the configured `SETTING.model` against the list of locally-downloaded models. If it's missing, the run is aborted before any Python process spawns, and a themed `confirmDialog` explains what's happening:

  > **尚未下載模型 {{model}}**
  >
  > 你在設定中選擇的模型「{{model}}」還沒下載到本機。若直接執行轉錄，App 會嘗試即時從 HuggingFace 下載（大約數百 MB 到 1 GB，視模型大小而定），這段期間看起來會像卡住。建議先到「模型」分頁下載後再回來開始轉錄。

  Clicking **前往「模型」下載 / Go to Models** switches the renderer to the Models tab and raises a toast that tells the user which entry to click and that they can return to the Main tab when the download finishes. Clicking **取消 / Cancel** just aborts the run.

  The check is wrapped in `try / catch` so a failing `listModels` call (venv not yet bootstrapped, IPC error, etc.) falls through silently and lets the existing Python error path surface whatever the real problem is — same behaviour as before the guard was added.

## Migration notes

No user action required. If you were previously running transcription without downloading the model first, v1.7.12 will catch you the next time you try and offer to redirect you to the Models tab instead. If the model is already downloaded, the guard is a single `listModels` IPC call on click and adds no perceptible latency.
