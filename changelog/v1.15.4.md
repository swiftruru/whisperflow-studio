# v1.15.4

Patch release fixing a gap in v1.15.3's drag-and-drop fix: the media-folder path still wouldn't update when the dropped file was already in the queue (or otherwise skipped by `addFiles()`).

## Fixed

- **Media-folder path updates on every drop, not just successful adds** ([src/renderer/index.js](src/renderer/index.js)) — dropping a file onto the media-folder card now calls `applyDirectory(parentDir)` *before* `addQueueFiles()`, so the folder UI reflects the user's intent even when the file is already queued, already has subtitles, or is an unsupported format. The same ordering applies to the OS file-association bridge (Finder "Open with").
