# v1.7.2

Patch release that fixes a **major packaging bug**: the Settings tab's translation file has been silently excluded from every CI-built release since i18n was introduced, so the whole Settings tab was rendering in English HTML fallbacks instead of the real localized strings.

## Fixed

- **Settings-tab i18n missing from every packaged release** — the project-root `.gitignore` had an un-anchored `settings.json` rule (`settings.json` instead of `/settings.json`), which matched **every** file named `settings.json` anywhere in the repo — including the two translation files `locales/en/settings.json` and `locales/zh-TW/settings.json`. Those files existed locally (untracked) so `npm run dev` always worked, but `actions/checkout` on CI never pulled them, `electron-builder` never bundled them into the asar, and every DMG / EXE / AppImage shipped since v1.5.0 had a completely empty `settings` namespace. The runtime `t('settings:*')` calls returned empty strings, the renderer fell back to each element's HTML default text, and users on packaged builds saw:
  - "Config Settings" (CSS-uppercased to "CONFIG SETTINGS") instead of 設定 / Settings
  - "Save" instead of 儲存 / Save (the HTML fallback happens to match the EN translation so this one was invisible)
  - No field descriptions at all under any config input (because `t('settings:fields.<key>.description')` returned empty → the component skipped rendering the description element entirely)
  - The pythonPath "Detect now" button squinted into a visually empty pill in zh-TW because the label fell through the i18n wrapper empty-string safety net
  The fix anchors the rule to the project root (`/settings.json`) so only the portable dev override is ignored, and commits the two translation files that should have been tracked from day one.
- **Legacy `poetryPath` setting field removed** — earlier versions wrote `poetryPath` to `settings.json` for the old Poetry-based bootstrap path. Since v1.4.0 the bundled venv path alone (`pythonPath`) is sufficient, but the legacy key kept living in users' `settings.json` and was still rendered as a form field on the Settings tab. `normalizeLocalSettings` now strips any known legacy key on load and re-writes the file so the cleanup is persistent.
- **System Check row layout (retouch)** — a hotfix in v1.7.1 moved the preflight row's action button onto its own second row via CSS grid; v1.7.2 tightens the spacing and right-alignment (no functional change).
- **pythonPath "Detect now" button visibility** — even with the i18n fix landing, the old button styling (`font-size: 10px`, dim gold-on-cream, ~60px wide) was easy to mistake for an empty pill when the label was only four Chinese characters. Bumped to `font-size: 12px`, `font-weight: 700`, stronger `amber` foreground, and slightly larger padding so the label reads cleanly in both zh-TW and en.

## Migration notes

No user action required. The next launch of v1.7.2 will silently re-write `settings.json` to drop the legacy `poetryPath` key, and the Settings tab will finally show the full bilingual UI it was supposed to have all along.
