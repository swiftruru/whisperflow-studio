# v1.7.3

Patch release that renames release artifacts to include an explicit `mac` / `win` / `linux` segment, so users landing on the GitHub Releases page can tell the platform of each asset at a glance without having to infer it from the file extension.

## Changed

- **Release artifact filenames now carry a platform segment** — previously the built installers were named `WhisperFlow-Studio-1.7.2-arm64.dmg`, `WhisperFlow-Studio-Setup-1.7.2.exe`, and `WhisperFlow-Studio-1.7.2.AppImage`, which required the user to recognize DMG/EXE/AppImage extensions to know which one to pick. The new filenames are:
  - `WhisperFlow-Studio-1.7.3-mac-arm64.dmg`
  - `WhisperFlow-Studio-1.7.3-mac-x64.dmg`
  - `WhisperFlow-Studio-1.7.3-win-x64.exe`
  - `WhisperFlow-Studio-1.7.3-linux-x86_64.AppImage`
  The stable version-less aliases generated by the release workflow follow the same scheme (`WhisperFlow-Studio-mac-arm64.dmg`, `WhisperFlow-Studio-win-x64.exe`, etc.) so the README download badges and in-app auto-update still resolve against deterministic URLs.
- **README download badges and the bilingual release-download-guide** — updated to reference the new stable-alias filenames. The guide's `__VERSION__` placeholder is still substituted at workflow time so each release shows its own version in the filename example.

## Migration notes

No user action required. Windows users who installed v1.7.2 will continue to receive auto-updates through the existing `latest.yml` metadata — the new filenames in `latest.yml` are still picked up correctly by `electron-updater` because the metadata is regenerated on every build. macOS and Linux users following the README download badges are unaffected because the stable-alias URLs point at the new filenames.
