# v1.15.5

Queue semantics refactor and media-folder card polish.

## Changed

- **Scan now appends instead of overwriting** ([src/main/queue-manager.js](src/main/queue-manager.js)) — scanning a folder keeps existing pending / running / paused / failed jobs and only adds files not already queued. Completed (`done` / `skipped`) jobs are auto-dropped at the start of each scan so the queue no longer accumulates indefinitely.

- **Scan summary reports specific outcomes** ([src/main/queue-manager.js](src/main/queue-manager.js), [src/main/ipc-handlers.js](src/main/ipc-handlers.js), [locales/*/events.json](locales/zh-TW/events.json)) — `scanSummary` now tracks `newlyAdded`, `alreadyQueued`, `filteredBySubtitle`. The console log emits a new `scanAppendSummary` line like "新增 5 個、已在佇列 3 個、已有字幕 12 個".

- **Neutral toast wording for skipped drops** ([locales/*/queue.json](locales/zh-TW/queue.json)) — dropped files that don't enter the queue now show a descriptive reason ("此檔案已在待轉錄清單中" / "此檔案同資料夾內已有字幕") instead of the accusatory "跳過：" prefix.

- **"Scanned but all have subtitles" is no longer misreported as "no files found"** ([src/renderer/components/queue-panel.js](src/renderer/components/queue-panel.js), [locales/*/progress.json](locales/zh-TW/progress.json)) — the batch card now shows "掃描到 N 個檔案，皆已有字幕" when everything the user scanned was filtered out, instead of the misleading "找不到缺字幕的檔案".

- **Recent folders moved into a dropdown next to Browse…** ([src/renderer/index.html](src/renderer/index.html), [src/renderer/index.js](src/renderer/index.js), [src/renderer/styles.css](src/renderer/styles.css), [locales/*/queue.json](locales/zh-TW/queue.json)) — the vertical "recent directories" list used to eat ~5 lines of the main column even when rarely used. It's now collapsed into a `▾` popover beside the Browse button (only shown when there are entries).

- **Recent folders auto-prune stale paths** ([src/main/ipc-handlers.js](src/main/ipc-handlers.js), [preload/preload.js](preload/preload.js), [src/renderer/index.js](src/renderer/index.js)) — new `fs:filter-existing-dirs` IPC lets the renderer drop any recent entry whose folder no longer exists (opening the popover and at app start), so users can't pick a dead path and trigger a preflight error.
