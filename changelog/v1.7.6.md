# v1.7.6

Patch release that improves the Windows ffmpeg install UX by preferring **winget** over **Scoop**, and warning users up front when they do pick Scoop.

## Changed

- **Install-ffmpeg dialog now lists winget before Scoop on Windows** — Windows 10 / 11 ships with winget pre-installed (App Installer), so it's almost guaranteed to be available without the user having bootstrapped Scoop themselves. winget's `install` also hits exactly one package and typically finishes in under a minute, versus `scoop install <pkg>` which first runs a full `scoop update` across every installed bucket (main, extras, java, versions, …) and on a user with many apps can take 5–15 minutes before ffmpeg itself even starts downloading. Chocolatey continues to be listed last because its install path requires admin.
- **Scoop row now shows a bilingual up-front warning** — when Scoop is detected as available, the install dialog renders an inline hint under the Scoop status line:
  > ⚠ The first Scoop install runs a full bucket update first — this can take several minutes before ffmpeg itself starts downloading.
  > ⚠ 首次使用 Scoop 會先檢查所有 bucket 更新，可能需要數分鐘；之後才會開始下載 ffmpeg 本體。
  No other package manager has this behaviour, so the hint only renders on Scoop rows. Users who still prefer Scoop now know to wait through the update check instead of thinking the install is hung.

## Migration notes

No user action required. No settings or data touched.
