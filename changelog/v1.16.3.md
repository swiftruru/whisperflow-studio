# v1.16.3

修正 large-v3 模型下載 / 載入卡關、切換媒體資料夾時舊清單殘留、以及佇列卡標頭中文字「跑直」的版面問題。

## Fixed

- **large-v3 模型下載與「未下載」誤判** ([python/whisperflow/models/manager.py](python/whisperflow/models/manager.py), [src/main/ipc-handlers.js](src/main/ipc-handlers.js)) — 下載驗證硬性檢查 `vocabulary.txt`，但 `Systran/faster-whisper-large-v3` 改用 `vocabulary.json`，導致下載完成卻被判失敗、即使 large-v3 已成功安裝、按下「開始轉錄」也會跳出「尚未下載模型」對話框。改為兩個檔名擇一即可（與 `model.bin` / `model.safetensors` 同樣的處理方式），Python 端與 main 端 guard 同步修正
- **切換媒體資料夾後舊資料夾的待處理項目殘留** ([src/main/queue-manager.js](src/main/queue-manager.js)) — `scanMedia` 是 append 模式，會保留所有未完成 jobs；切換到新資料夾再掃描時，舊資料夾的 pending / failed 項目仍留在佇列裡。現在掃描開始前會剔除掉 `filePath` 不在新 `rootPath` 下的被動項目，但仍無條件保留 `running` / `paused` 的 active job，避免中斷正在進行的工作
- **佇列卡標頭在中文 + 雙按鈕時被擠成直書** ([src/renderer/styles.css](src/renderer/styles.css)) — 「佇列」label、「N 個檔案」badge、「重試失敗項目」/「清除已完成」按鈕同時放進 `.card-label` flex row，左欄寬度不足時 CJK 字元因無斷字規則被一字一行往下排。`.queue-card .card-label` 加上 `flex-wrap: wrap` 讓動作按鈕在空間不足時自動換行，並對 label / badge 套 `white-space: nowrap` 防止 CJK 逐字折行
