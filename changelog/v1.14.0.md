# v1.14.0

Adds an in-app **subtitle editor** so typos and recognition errors can be fixed without leaving the app. Pick any finished transcription from the preview or history, edit the text in a themed table-style modal, and hit Save — the editor rewrites every subtitle file you have enabled in Settings (`.srt` / `.vtt` / `.txt` / `.json`), preserving the timeline and all the JSON side-channel fields Whisper produces. Includes find-and-replace, IME-safe keyboard handling, and a tmp-dir backup on every save so your original files are never clobbered in place.

## Added

- **Subtitle editor modal** ([src/renderer/components/subtitle-editor.js](src/renderer/components/subtitle-editor.js)) — reuses the transcript-preview's full-screen modal skin and shows each segment as `# / time (read-only) / text (editable)`. Timestamps intentionally stay locked because mis-timed subtitles are far harder to spot than typos. Launch from the new "編輯字幕 / Edit subtitles" button in the transcript preview footer, or the pencil icon on any successful row in the Recent Transcripts history.
- **Multi-format save pipeline** ([src/main/subtitle-writer.js](src/main/subtitle-writer.js)) — a single Save rewrites every format you have enabled in Settings → Subtitle output. SRT and VTT are regenerated from the edited segments; TXT mirrors Python's `write_txt` (one stripped segment per line); JSON is patched in place so `language`, `duration`, per-segment `id` / `avg_logprob` and any other fields Whisper wrote stay untouched — only the `text` is updated. Formats that are enabled in Settings but have never been generated on disk are skipped (we never fabricate a fresh file behind your back).
- **Tmp-dir backups instead of `.bak` beside the media** — before every overwrite, originals are copied to `os.tmpdir()/whisperflow-studio-backups/<ISO-timestamp>/`. Your media folders stay clean and you can still grab the previous version if an edit needs undoing.
- **Find & replace** — Cmd/Ctrl+F toggles a two-row search bar matching the editor's theme; Enter / Shift+Enter jumps to the next / previous match, Replace or Replace All rewrites matches in place, and a `current / total` counter tracks progress. All match navigation focuses the owning textarea and highlights the range so you see exactly what's about to change.
- **`subtitle:save` IPC** — new handler exposed via `window.electronAPI.transcript.save({ mediaPath, outputDir, segments, formats })`. Returns `{ written, skipped, backupDir, backups, bytes }` (or a structured error) so the renderer can give a precise toast without guessing.

## Changed

- **Transcript preview footer has a new "編輯字幕" action** between Reveal-in-folder and Close, so the edit path is discoverable without a second click.
- **History rows get a pencil edit button** next to the existing eye / folder icons when a transcript file is on disk.
- **All editor / preview confirm dialogs use the themed `confirmDialog`** instead of the OS-native `window.confirm`, so the cream/amber palette is preserved and Enter/Esc/backdrop behaviour matches the rest of the app.
- **Subtitle segments are single-line** — Enter is swallowed in the editor textareas (except during IME composition, which still commits candidates normally) and any pasted multi-line text is collapsed to a space, so edits can never produce stray multi-line SRT segments.

## Fixed

- **Revert used to discard your edits to the last *saved* state, not the session-start state** — `state.original` was being overwritten after each successful save, so pressing Revert gave the illusion of "nothing happened". Original now stays frozen from the moment the editor opens; Revert always undoes back to that, even across multiple saves. Save / Revert buttons now use independent predicates (`dirty` for Save, `draft != session-start` for Revert).
- **Backdrop clicks no longer close the subtitle editor or the transcript preview** — dragging a text selection out of a textarea could unintentionally land a click on the dim overlay, silently tearing down the editor mid-edit. Close is now restricted to the ✕ button, the Close button, or Esc.
- **Escape during IME composition no longer closes the editor** — 注音/Pinyin/Kana IMEs use Esc to cancel an in-progress candidate; the global keydown handler now bails early when `event.isComposing` is set.
- **Enter inside the find / replace boxes no longer fires mid-IME** — the same composition guard covers the find bar so pressing Enter to commit a 注音 candidate doesn't prematurely trigger Replace.
- **Save toast no longer wraps to four lines on long filenames** — collapsed to `已更新 N 個字幕檔（SRT / VTT / …）` with an optional skipped-formats tail, instead of listing every absolute path.
- **Confirm dialog layout polish** — fixed title/message left-edge alignment, gave the action row a consistent `min-width` / `white-space: nowrap` so long Chinese button labels like "放棄並關閉" sit inside the dialog border instead of visually spilling to the right.

## Migration notes

- No user action required. The editor is additive — existing transcripts and config files keep working as before, and the editor only touches a subtitle file when you explicitly hit Save. If you previously relied on a `.bak` file appearing next to your media, note that backups now live in the OS temp directory; the path is surfaced in the save IPC result for callers that need it.
