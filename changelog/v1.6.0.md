# v1.6.0

Minor release that adds a dedicated **About tab** and replaces the dark theme with a new **Cocoa Cream** palette designed to feel like the same app as the cream-yellow light theme, just with the lights turned down.

## Added

### About tab

A new fourth tab that acts as WhisperFlow Studio's in-app product page. It includes:

- **Hero block** — app icon, title, and a live version badge that reads from `package.json` at runtime via a new `app:get-version` IPC handler, so every release automatically shows the correct version without touching a single constant in the UI.
- **Author card** — circular monogram avatar ("RU") with a yellow-to-peach gradient, author name (潘昱如 / Yu-Ru Pan), role description, and two external-link buttons (GitHub repository and personal site) that go through the sandboxed `shell:open-external` IPC.
- **Tech stack card** — six feature-oriented groups (Desktop Shell, Transcription Core, Python Runtime, UI & Styling, Internationalization, Packaging) with 19 items total, each bullet decorated with a cycling accent dot so the list feels structured without needing a table.
- **Credits & License card** — MIT notice, inline acknowledgement of the upstream projects (faster-whisper-webui, faster-whisper, CTranslate2, Silero VAD), and two inline-pill action buttons linking out to the full `NOTICES.md` and the GitHub issue tracker.
- **Footer** — copyright line as a quiet visual close to the page.

The entire About tab is fully bilingual. A new `about` namespace in the i18n resources ships with 42 keys per locale, covering every label, description, button, and toast message. The titlebar language toggle flips the whole tab live without a reload.

### Dark theme: Cocoa Cream

The previous warm-brown dark theme read as "old sepia paper" and cool attempts read as "database terminal". The new palette is a soft mid-dark warm-neutral, closer to a "café au lait" than either extreme, with clean cream-white text and borders that keep the cream-yellow brand accent tonally native. Key changes versus the old dark theme:

- **Surfaces lifted to the 14–28% lightness range** (six clear steps) instead of the old saturated brown `#242018` canvas. Cards now visibly lift from the sidebar background without the whole thing looking like vintage paper.
- **Text is warm cream-white** (`#fbf5e6`) at roughly 13:1 contrast on the canvas, with every subtext tier meeting WCAG AA. No more unreadable card labels or About-page descriptions.
- **Borders use cream-white alpha** (`rgba(255,240,200,0.14)`) instead of the old yellow tint, so card edges no longer carry a visible yellow cast.
- **Shadows and brand glow** tuned softer because the canvas isn't near-black anymore — primary buttons and focus rings still glow but the effect feels like lamplight rather than a floodlamp in a cave.
- **Status colours** (`--green`, `--red`, `--blue`, `--peach`) lightly desaturated so they sit comfortably on the warm canvas without screaming.

The light theme is completely untouched.

## Changed

- **`src/main/ipc-handlers.js`** — new `app:get-version` handler that returns `app.getVersion()` directly, so any renderer component can read the current version without hardcoding a constant.
- **`preload/preload.js`** — exposes `getAppVersion()` on `window.electronAPI` for the About page to consume.
- **`src/main/i18n.js`** — the `NAMESPACES` list now includes `about`, making it the 14th feature namespace.
- **`electron-builder.yml` + `.github/workflows/release.yml`** — artifact names are now versioned (for example `WhisperFlow-Studio-1.6.0-arm64.dmg`) while the release workflow also uploads stable version-less aliases so the README's `releases/latest/download/...` buttons keep working across releases. v1.6.0 is the first release where this versioned-plus-alias scheme takes effect, so the Assets list should show both names side by side.

## Fixed

- **Settings form row spacing** from v1.5.0 — left untouched in this release. Carried over from the previous dark theme polish.
- **Card label readability** in dark mode — the old dark theme had `--overlay0` too dim for card labels ("MEDIA DIRECTORY", "BATCH PROGRESS", etc.). The Cocoa Cream palette gives `--overlay0` a proper prominent-gold value (`#bdab7a`) so card labels read clearly again.

## Migration notes

No user action required. Reinstall v1.6.0 over the existing install; the bundled venv, downloaded models, and your saved settings (including `uiLanguage` and theme preference) are all preserved. If you previously liked the warm-brown dark theme and want it back, the old palette is still viewable in the git history at `v1.5.0`, but the feedback that drove this change (unreadable card labels, sepia vibe, brand discontinuity) is not expected to come back.
