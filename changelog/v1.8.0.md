# v1.8.0

Minor release that replaces the frozen **"下載中…"** label with a full **model download manager** — real-time progress bar, speed, ETA, cancel, retry, and a titlebar indicator so you always know what's happening.

## Added

### Model download progress system

Previously, clicking "Download" on a model in the Models tab fired a blocking IPC that showed nothing but a static "下載中…" button for the duration of the download (up to 15+ minutes for `large-v2` at 2.8 GB). Users reasonably assumed the app was hung. v1.8.0 replaces this with a full streaming pipeline:

- **Persistent download card on the Models tab** — appears the moment a download starts and shows:
  - Model name and current stage (importing from cache / downloading / verifying)
  - Progress bar with percentage
  - Downloaded bytes / total bytes (e.g. `1.24 GB / 2.82 GB`)
  - Download speed (e.g. `15.3 MB/s`)
  - Estimated time remaining (e.g. `ETA 1m 42s`)
  - Cancel button that kills the download child process (including the full Windows process tree via `taskkill /T /F`)
- **Titlebar download indicator** — a small pulsing amber chip in the titlebar (next to the language toggle) that shows the current percentage while a download is running. Visible from every tab so you don't have to stay on the Models tab to monitor progress. Click it to jump back to the Models tab.
- **Download history** — completed, failed, cancelled, and interrupted downloads are shown in a collapsible list below the progress card, each with a Retry button so you can re-attempt a failed download without scrolling through the model list.
- **Cancel + retry** — cancel mid-download at any time; retry picks up where it left off because `huggingface_hub`'s built-in ETag/blob resume mechanism automatically skips already-downloaded chunks. This is effectively free resume support without any custom range-header management.
- **State persistence** — download state is saved to `userData/download-state.json` (debounced + synchronous before-quit flush). If the app crashes or is force-closed mid-download, the next launch marks the interrupted entry as "interrupted" and shows a Retry button. Retrying resumes the partial download.
- **`downloads` i18n namespace** — ~36 keys per locale covering the card, toasts, stage labels, status badges, error messages, and action buttons. Brings the total to 17 namespaces with 602 keys per locale.

### Architecture

The download pipeline is a new independent chain that runs alongside the transcription runner without interfering:

- **Python side**: `ModelManager.download()` now calls `huggingface_hub.snapshot_download()` directly (bypassing `faster_whisper.download_model()` which hardcoded `tqdm_class=disabled_tqdm`) with a custom `WhisperFlowProgressTqdm` subclass that captures per-blob byte deltas into a `SharedProgressState` accumulator. The accumulator computes a 10-second sliding-window speed + ETA and emits `[WhisperFlowEvent]` JSON lines to stdout via the existing `events.py` infrastructure. The CLI's `--emit-events` flag activates the event stream; without it, `--download-model` behaves exactly as before (single JSON line at end) for backwards compatibility.
- **Main process**: `model-download-runner.js` spawns the Python CLI with `--emit-events`, pipes stdout through `createLineBuffer` + `parseRunnerEventLine` (reusing the transcription runner's event parser), and mutates `download-state.js` (the single source of truth) on every event. `download-state.js` broadcasts `downloads:state-updated` to all renderer windows and persists to disk.
- **Renderer**: `download-state.js` (observer store mirroring `queue-state.js`), `download-panel.js` (persistent card), and `download-indicator.js` (titlebar chip) all subscribe to the same state and re-render on every tick.

## Changed

- **Default theme is now always light on first launch** — previously the app followed the OS dark-mode preference on first boot, which meant users on dark-mode systems saw the Cocoa Cream dark palette as their first impression instead of the primary cream-yellow design. The in-app theme toggle still works; once the user flips to dark mode, `localStorage` remembers and subsequent launches honour the choice. `BrowserWindow.backgroundColor` and the Windows `titleBarOverlay` colours are also set to light cream unconditionally so there's no dark flash on boot.
- **`model-manager.js :: handleDownload()`** — no longer awaits the blocking `downloadModel` IPC. Instead calls `window.electronAPI.downloads.start(name)` (fire-and-forget) and lets the download-panel component handle all progress feedback.
- **`python-runner.js`** — exports `createLineBuffer` so the new download runner can reuse it without duplicating the line-buffering logic.
- **`ipc-handlers.js`** — new `downloads:start / cancel / retry / clear-history / get-state` handlers. The old `models:download` handler is preserved as a legacy shim.

## Fixed

- **Unit test for `test_download_raises_when_no_cache_and_network_disabled`** — the test was mocking `faster_whisper.download_model` which we no longer call. Updated to mock `huggingface_hub.snapshot_download` instead.

## Migration notes

No user action required. The old `models:download` IPC still works as a legacy shim, so any external scripting against it is unaffected. The new `download-state.json` file is created automatically in `userData/` on first download.
