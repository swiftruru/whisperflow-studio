# v1.5.0

Major release: WhisperFlow Studio is now a fully bilingual app with a
production-grade i18n architecture, improved Settings UX, a sharper UI
contrast pass, and a one-click Python detect button.

## Added

### Production-grade i18n (Traditional Chinese / English)

- **New [locales/](locales/) tree** — 13 namespaces per locale
  (`common`, `sidebar`, `preflight`, `settings`, `queue`, `progress`,
  `models`, `console`, `controls`, `dialogs`, `errors`, `events`,
  `toasts`), ~370 keys per language, split by feature area rather
  than by page so shared components only need to register once.
- **i18next core** shared between the main process and the renderer
  via a single JSON source of truth.  Main process hydrates its own
  instance for Electron native dialogs
  (`dialog.showMessageBox(...)`); renderer gets its own instance via
  a one-shot `i18n:get-initial` IPC so `t()` is synchronous.
- **Titlebar language toggle** — new `中 / EN` pill button next to
  the theme toggle.  Clicking flips the app language live, no
  reload: static HTML (via `data-i18n` walker), dynamic components
  (via `app:language-changed` listener), main-process error
  banners, and Python runner stage messages all re-render in the
  new locale within ~100ms.
- **Auto-detect on first launch** — reads `app.getLocale()` and
  picks `zh-TW` for any `zh-*` system, `en` for `en-*`, `zh-TW` as
  the fallback for unknown locales.  User preference is persisted
  to `settings.json :: uiLanguage` and honoured on subsequent
  launches.
- **Key-based main → renderer contract** — `createAppError` /
  `createPreflightCheck` / `events.py :: emit` all carry
  `titleKey` / `messageKey` (+ `*Params`) alongside a legacy string
  fallback.  The renderer translates at display time, so switching
  language updates already-visible error banners and preflight
  checks without having to re-trigger them.
- **DOM binding via data-attributes** — static HTML uses
  `data-i18n="ns:key"` / `data-i18n-attr="placeholder=ns:key"`
  processed by a small 80-line walker
  ([`src/renderer/lib/i18n-dom.js`](src/renderer/lib/i18n-dom.js)).
- **Python stage messages are now i18n-aware** —
  [`events.py`](python/whisperflow/events.py) ships `messageKey` +
  `messageParams` fields; [`transcriber.py`](python/whisperflow/transcriber.py)
  emits stable keys (`events:stage.preparing`,
  `events:stage.loadingModel`, etc.) and the renderer localizes on
  receipt.  Python itself stays English end-to-end — the key lookup
  happens in JavaScript.
- **i18n-lint in CI** —
  [`scripts/i18n-lint.js`](scripts/i18n-lint.js) walks both locale
  trees, flattens each namespace, and fails the build if any key is
  present in one locale but not the other, or if a key is a leaf
  in one locale and a branch in another.  Wired into
  `npm run build:*` via the `prebuild:*` hooks and the
  `.github/workflows/release.yml` pipeline.
- **Per-parameter descriptions in Settings** — every config knob
  (`model`, `device`, `compute_type`, `vad`, `vad_merge_window`,
  `vad_max_merge_size`, `vad_padding`, `vad_prompt_window`,
  `language`, `initial_prompt`, `initial_prompt_mode`,
  `media_root_path`, `pythonPath`, …) now ships with a concise
  inline description in both languages so non-expert users
  understand what each setting actually does.

### UI polish

- **Contrast pass on light / dark themes** — cream-background tokens
  `--overlay0`, `--overlay1`, `--subtext0`, `--subtext1` were all
  too pale on the cream background.  They're now tuned to roughly
  AA contrast so card labels ("MEDIA DIRECTORY", "BATCH PROGRESS"),
  tab buttons, status badges, and dashed section dividers are
  actually readable.
- **Redesigned Settings form rows** — each field-row now has proper
  vertical padding, a dashed bottom divider, and `align-items: start`
  so the label pins to the top-left corner instead of floating in
  the middle of the row when the description is tall.
- **Per-field description callouts** — the new description blocks
  render as left-bordered cream-yellow boxes anchored to their
  input, clearly visually belonging to the setting above them
  instead of running into the next row.
- **"Detect now" button next to `pythonPath`** — explicit one-click
  trigger to re-run `resolveSystemPython()` and auto-fill the field
  with whatever the detector found, then refresh the main-tab
  preflight so the "找不到 Python 3" banner clears in the same
  action.  Useful when the user installed Python *after* launching
  the app and needs the resolver to re-discover it without
  restarting.

## Changed

- **`settings.json :: uiLanguage` is no longer rendered as a form
  field.**  The titlebar toggle is the single source of truth; the
  Settings form's `splitFormValues()` preserves the key on save so
  `writeAppSettings` never drops it.
- **Queue panel stage / status labels** now come from
  `progress:stage.*` / `progress:jobStatus.*` keys, so the Batch
  Progress chip and queue item status pills both respect the
  current language.
- **History "time ago" formatter** now uses localized keys
  (`progress:history.justNow`, `progress:history.minutesAgo`, …)
  instead of hardcoded strings.  Re-renders live on language
  switch.
- **Model Manager descriptions** are translated via
  `models:registry.<name>.description` keys with i18next's
  `defaultValue` option falling back to whatever Python's
  `ModelEntry.description` field said, so newly-added models still
  render *something* until the translation catches up.
- **[electron-builder.yml](electron-builder.yml)** now includes
  `locales/**` in the packaged file set.

## Fixed

- Error banner / error dialog titles used to read `執行失敗` when
  the underlying error already carried a better title from the
  main process.  The key-based payload shape + renderer-side
  localization fix this.
- Preflight internal error states (`preflight_internal_error`,
  `preflight_render_error`, `preflight_stuck_timeout`) are now
  properly localized — previously they were pinned to Traditional
  Chinese strings inside `preflight-panel.js`.

## Migration notes

- No user action required.  Reinstall v1.5.0 over the existing
  install; the bundled venv, downloaded models, and your saved
  settings are all preserved.
- On first launch after upgrading, the app reads
  `app.getLocale()` for any new install where `uiLanguage` isn't
  set — existing installs keep whatever's already in
  `settings.json`.
- If you've built a custom translation for your own locale,
  duplicate `locales/en/` to `locales/<your-lang>/`, add the
  language code to `SUPPORTED_LANGUAGES` in
  [`src/main/i18n.js`](src/main/i18n.js), and run
  `npm run i18n:lint` to verify the key set is balanced.
