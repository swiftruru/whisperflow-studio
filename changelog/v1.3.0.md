# v1.3.0 (Draft)

## Batch Workflow
- Replace the old single-file scan flow with a queue-based batch workflow
- One scan now builds the full queue of media files without subtitles
- Add a Batch Progress card and Queue list so the current file, processed counts, and queue state are always visible

## Runtime Controls
- Add `Pause` / `Resume` controls for the current transcription job
- Add `Skip Current` to move past the current file without marking the whole batch as failed
- Keep `Stop Batch` as a hard stop for the active process while preserving the remaining queue state
- Improve auto-loop so it continues through the queued files after a single scan

## Progress and Timing
- Add structured runner events from the Python bridge instead of relying only on stdout keyword matching
- Forward runner events through Electron IPC so queue state can react to formal stages such as `loading-model`, `transcribing`, and `writing-subtitle`
- Add current-job elapsed time and ETA to the Batch Progress card
- Add batch-level elapsed time and ETA estimates derived from current job progress and queued work
- Surface per-job stage messages and timing details in the Queue panel

## Preflight and Settings UX
- Add a startup System Check panel for Poetry, media root, tool paths, and bridge scripts
- Allow `Scan` with only a valid media root while still requiring full preflight for `Run Transcription`
- Show the resolved Poetry path directly in Settings even when it is coming from auto-detection

## Internal Changes
- Add `queue-manager.js` as the main-process owner of queue state and batch lifecycle
- Add queue state IPC and renderer-side queue store/panel components
- Extend the Python runner to support pause/resume/stop control for the spawned subprocess
- Add `runner-event.js` and `runner-metrics.js` for structured stage parsing and ETA helpers

## Docs
- Rewrite README workflow docs around the queue-based batch model
- Update project structure docs for the new preflight, queue, and runner event components
