# v1.16.2

Fixes the CUDA fallback warning from v1.16.0 so it respects the UI language and is prefixed/classified like every other backend log line.

## Fixed

- **CUDA-fallback warning now translates + gets the `[Python]` prefix and warn colour** ([python/whisperflow/models/device_probe.py](python/whisperflow/models/device_probe.py), [src/main/python-runner.js](src/main/python-runner.js), [locales/*/events.json](locales/zh-TW/events.json)) — v1.16.0 hardcoded the fallback message as Chinese inside Python, so English-mode users saw the raw zh-TW string and it slipped past the main-process rewrite layer (no `[Python]` prefix, no colour classification). The Python side now emits a stable English log `CUDA runtime unavailable, falling back to device=cpu, compute_type=X (was compute_type=Y)`; `python-runner.js` matches it via the existing `LOG_REWRITE_PATTERNS` table and renders the localized `events:log.cudaFallbackToCpu` string (English + 繁中), prefixed with `[Python]` and starting with `Warning:` / `警告：` so the console classifier colours it as a warn line.
