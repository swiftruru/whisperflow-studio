# v1.15.0

A polish-heavy follow-up to the v1.14 subtitle editor: **undo / redo with keyboard shortcuts**, **find and replace**, and a pile of accessibility and accident-prevention fixes that shook out of real-world use. The editor now behaves like a real text editor — session-wide undo that survives saves, IME-safe keyboard handling, themed confirmation dialogs, and no more silent data loss from stray backdrop clicks or misfired Enter keys.

## Added

- **Undo / Redo with full history stack** ([src/renderer/components/subtitle-editor.js](src/renderer/components/subtitle-editor.js)) — a text-snapshot history (up to 200 entries) with 500ms debounced typing commits, immediate commits on Replace / Replace All / Revert, and a rolling `savedIndex` marker so saving a file does **not** erase history. You can save, keep editing, and still Cmd+Z back to any earlier state — even before the save. Custom SVG undo / redo buttons in the toolbar and keyboard shortcuts (⌘Z / ⌘⇧Z on macOS, Ctrl+Z / Ctrl+Y on Windows/Linux) override the browser's per-textarea native undo so a single keystroke reverts across segment boundaries.
- **Find & replace** — a collapsible two-row search bar inside the editor toggled by ⌘F / Ctrl+F. Enter jumps to the next match (Shift+Enter for previous), a live `current / total` counter tracks progress, and a case-sensitive toggle (`Aa`) filters matches. Replace works one-at-a-time or in bulk; the current match is auto-scrolled into view and highlighted inside the owning textarea so you can see exactly what's about to change.

## Changed

- **Session-start `original` is now immutable** — previously the editor overwrote `state.original` on every successful save, which made "Revert" meaningless after saving (it would "revert" to the just-saved state). The session-start snapshot is now frozen from the moment you open the editor, so Revert always restores to the on-disk content at open time, no matter how many saves happened in between.
- **Save and Revert buttons use independent predicates** — save button enables when `draft !== lastSaved`, revert enables when `draft !== sessionStart`. Previously they were both hooked to a single `dirty` flag, so after a save-then-edit flow the Revert button would go grey even when there were still edits to discard.
- **All confirm dialogs use the themed `confirmDialog`** instead of the OS-native `window.confirm`, so the cream / cocoa palette is preserved and Enter / Esc / backdrop behaviour matches the rest of the app.
- **Subtitle segments are strictly single-line** — Enter is swallowed in textareas except during IME composition (so 注音 / Pinyin candidate commits still work), and pasted multi-line content is collapsed to a single space on the fly. Existing multi-line segments loaded from disk are also flattened on open so they never round-trip as multi-line in the file.
- **Close button relabelled from "取消 / Cancel" to "關閉 / Close"** — the footer action closes the editor regardless of save state, so "Cancel" read as "cancel my save" to some users. "Close" is unambiguous.
- **Confirm dialog layout polish** — fixed title / message left-edge alignment, enforced `min-width` and `white-space: nowrap` on action buttons so long Chinese labels like "放棄並關閉" sit cleanly inside the dialog border.

## Fixed

- **Subtitle editor and transcript preview no longer close on stray backdrop clicks** — dragging a text selection out of a textarea and releasing over the dim overlay was firing a click on the backdrop and silently closing the editor, nuking unsaved edits. Close is now restricted to the ✕ button, the Close button, or Esc.
- **Escape during IME composition no longer closes the editor** — 注音 / Pinyin / Kana IMEs use Esc to cancel an in-progress candidate; the global keydown handler now bails early when `event.isComposing` is set.
- **Enter in the find / replace boxes no longer fires mid-IME** — same composition guard covers the find bar, so pressing Enter to commit a 注音 candidate in the Replace field doesn't prematurely run a Replace.
- **Revert requires confirmation and is disabled when nothing to revert** — previously one stray click on Revert (which was always enabled) could erase the entire session's edits without warning. Now it only lights up when `draft !== sessionStart`, and a destructive confirm dialog intercepts the click.
- **Save toast collapsed from four wrapped lines to one** — was listing every absolute file path written (`...SRT, …VTT, …TXT, …JSON`) which overflowed the toast. Now shows `已更新 N 個字幕檔（SRT / VTT / …）` with an optional skipped-formats tail.
- **Subtitle editor buttons now use `aria-disabled`** instead of the native `disabled` attribute on the undo, redo, save, revert, and find/replace controls. macOS Chromium maps the HTML `disabled` property onto the system-level `NSControl.isEnabled = NO`, which forces the OS-native not-allowed cursor and overrides any CSS cursor. Using `aria-disabled` keeps the control focusable-but-inert so the custom themed look stays consistent; click handlers bail when `aria-disabled="true"`, and Tab still skips disabled buttons via explicit `tabindex="-1"`.

## Migration notes

- No user action required. The editor UI adds buttons and a collapsible search bar but the transcript-preview → edit flow and the on-disk file format both stay the same.
- If you had a workflow around the `window.confirm`-style native dialog prompts, those are now replaced with themed modal equivalents — behaviour (Enter to confirm, Esc to cancel, backdrop click cancels) is unchanged.
