# v1.16.1

CI hotfix for v1.16.0.

## Fixed

- **`test_device_probe` no longer requires `faster_whisper` at collection time** ([python/whisperflow/models/device_probe.py](python/whisperflow/models/device_probe.py), [python/whisperflow/models/faster_whisper_backend.py](python/whisperflow/models/faster_whisper_backend.py), [python/whisperflow/tests/test_device_probe.py](python/whisperflow/tests/test_device_probe.py)) — extracted `_probe_cuda_runtime` and `resolve_device_and_compute_type` from the backend module into a new lightweight `device_probe.py` that has no `faster_whisper` import. The test now imports from there instead of from the backend module, so `pytest` collection succeeds on CI runners where the heavy ML dependencies are intentionally not installed. v1.16.0's build workflow failed on all three platforms for this reason.
