# v1.4.0

**Breaking change**: WhisperFlow Studio no longer requires an external `faster-whisper-webui` project or Poetry install. The transcription core has been rewritten directly into the app.

## Highlights

- **Self-contained transcription core** — new `python/whisperflow/` package drives faster-whisper, Silero VAD, segment merging, prompt strategies, and subtitle writers in-process. The old "Electron → bridge → subprocess into a separate Poetry project" chain is gone.
- **First-run venv bootstrap** — the app creates its own `python/.venv/` and runs `pip install -r requirements.txt` on first launch via the new System Check action button. Only Python 3.10+ on PATH is required.
- **Model Manager tab** — list / download / delete faster-whisper models with size hints and install state. Models go to an app-managed directory (`userData/models/`) instead of your global HuggingFace cache. Silero VAD weights share the same root (under `torch_hub/`).
- **Dynamic model dropdown** — the Settings tab's `model` field now lists only downloaded models, pulled live from the Model Manager. Falls back to the static catalogue if the venv isn't initialised yet.

## Added

- `python/whisperflow/` — ~20 rewritten modules covering transcriber, config, events, progress, languages, audio source, VAD (silero / periodic / parallel), models (registry / manager / cache / backends), prompts, and subtitle writers. All derived (rewritten, not copied) from [aadnk/faster-whisper-webui](https://gitlab.com/aadnk/faster-whisper-webui) under Apache 2.0.
- `python/whisperflow/tests/` — 46 pytest unit tests covering writers, segments, languages, prompts, events, model manager, and config.
- `python/requirements.txt` + `python/pyproject.toml` — bundled Python project metadata (faster-whisper, ctranslate2, torch, torchaudio, ffmpeg-python, more-itertools, intervaltree, srt, tqdm).
- `python/whisperflow/models/manager.py` — cross-platform managed models directory (matches Electron's `app.getPath('userData')` on every platform so CLI-direct and GUI-managed paths align):
  - macOS: `~/Library/Application Support/WhisperFlow Studio/models/`
  - Windows: `%APPDATA%/WhisperFlow Studio/models/`
  - Linux: `~/.config/WhisperFlow Studio/models/`
- `python/whisperflow/cli.py` — new entry point with `--list-models`, `--download-model NAME`, `--delete-model NAME` sub-commands.
- `src/main/venv-installer.js` — first-run venv creation + pip install with streaming progress.
- `src/renderer/components/model-manager.js` — new Models tab UI.
- IPC channels: `venv:status`, `venv:initialize`, `models:list`, `models:download`, `models:delete`.
- Preflight panel now surfaces warning-level checks (e.g. venv not initialised) with inline action buttons, not just errors.
- `NOTICES.md` — third-party attributions (Apache 2.0 for upstream, plus runtime dependency licenses).
- CI runs the whisperflow test suite on every platform before building the release artifact.

## Changed

- `bridge/run_cli.py` — from ~250 lines of subprocess wrapping into ~90 lines of direct `import whisperflow` adapter. No more double-subprocess chain.
- `src/main/path-resolver.js` — `resolvePoetryPath` → `resolveBundledPython` / `resolveSystemPython`.
- `src/main/python-runner.js` — spawns the bundled venv's Python directly instead of `poetry run python`.
- `src/main/preflight-checker.js` — validates the bundled `whisperflow` package and venv instead of an external tool path.
- `src/main/ipc-handlers.js` — drops `whisperToolPath` checks; injects a cross-platform `models_dir` into `config.json` on first load; adds venv/models IPC handlers.
- `python/config/config.json` + `config.example.json` — new schema. Removed: `whisper_faster_tool_path`, `whisper_implementation`, `python_executor`, `fp16_enabled`, `auto_parallel_enabled`, and the entire `WEBUI_SETTING` block. Added: `models_dir`, `device`, `compute_type`, `vad_merge_window`, `vad_padding`, `vad_prompt_window`. Renamed: `vad_argument` → `vad`, `vad_initial_prompt_mode` → `initial_prompt_mode`. The Python-side config dataclass accepts the old names as aliases so existing configs continue to load.
- `python/config/config.metadata.json` — updated enum options; replaced `knownPoetryPaths` with `knownPythonPaths` (three platforms); added `bundledPython` block (venv dir name, requirements file, minimum Python version); updated `pathFieldKeys` to reference `models_dir` / `pythonPath`.
- `settings.json` / `settings.example.json` — `poetryPath` → `pythonPath`.
- `src/renderer/components/settings-panel.js` — `poetryPath` validation renamed to `pythonPath`; `whisper_faster_tool_path` field no longer validated; the `model` dropdown now loads dynamically from `models:list`.
- `electron-builder.yml` — ships `NOTICES.md`; filters out `.venv`, `.venv-test`, `__pycache__`, `.pytest_cache` from the extraResources bundle.
- `.github/workflows/release.yml` — adds `actions/setup-python@v5` and runs `pytest whisperflow/tests/` before building.
- `package.json` — bumped to 1.4.0; description updated.

## Removed

- `_faster-whisper-webui/` — the vendored upstream project tree is gone. All required logic has been rewritten into `python/whisperflow/`.
- `python/faster-whisper-webui-cli.run.py` — replaced by the direct `import whisperflow.cli` path in `bridge/run_cli.py`.
- External Poetry dependency — the app no longer calls `poetry run python` anywhere.
- `whisper_faster_tool_path` / `whisper_implementation` / `python_executor` / `fp16_enabled` / `auto_parallel_enabled` settings.
- `WEBUI_SETTING` section in `config.json`.
- Error codes `MISSING_WHISPER_TOOL_PATH`, `INVALID_WHISPER_TOOL_PATH`, `INVALID_WHISPER_TOOL_PROJECT`, `INVALID_POETRY_PATH`, `POETRY_NOT_FOUND` — replaced by `BUNDLED_PYTHON_NOT_FOUND`, `VENV_NOT_INITIALIZED`, `VENV_INIT_FAILED`, `WHISPERFLOW_PACKAGE_MISSING`.
- Diarization, YouTube download, HuggingFace model conversion, alternative whisper backends (openai/whisper, dummy) — none were previously exposed in the UI and all were removed from the rewritten core to keep the codebase focused.

## Migration notes

- **You no longer need `faster-whisper-webui` installed separately.** Delete the `whisper_faster_tool_path` setting from your `config.json` (or let v1.4.0's schema migration drop it for you).
- **You no longer need Poetry.** Rename `poetryPath` in `settings.json` to `pythonPath` if you had a custom value.
- **You need Python 3.10+ on PATH** the first time you run v1.4.0. Click **立即建立環境** in the System Check panel to create the bundled venv and install dependencies.
- **Your downloaded models will re-download.** The managed models directory is new in v1.4.0 and doesn't touch your existing `~/.cache/huggingface/` cache. If you want to reuse your old weights, manually copy them into the managed directory (shown in the Models tab header) under the `models--Systran--faster-whisper-<name>` layout.
