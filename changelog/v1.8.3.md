# v1.8.3

Patch release with several fixes accumulated since v1.8.2.

## Fixed

- **Model-missing guard now blocks all transcription paths** — the guard was moved to the main-process `run:cli` handler so it covers button click, auto-loop, queue retry, and skip-and-continue. Previously only the initial button click was guarded.
- **False "transcription complete" notification** — `run:done(0)` could arrive from a prior scan while `lastAction` had been flipped to `'cli'`, making the handler think a transcription succeeded. Toast and system notification now only fire when `lastFinishedJob` actually exists.
- **Model-missing console log was hardcoded English** — now uses the main-process `t()` function with a localized i18n key so the message respects the UI language setting.
- **Download buttons stayed clickable during an active download** — clicking a second download button triggered an "already running" error toast. All download buttons are now disabled while any download is in progress; the active model's button shows "Downloading..." while others show their normal label but remain greyed out.
- **Model list not refreshing after download completes** — `refreshModelManager()` was inside `renderCurrent(dl)` which returned early when `dl` was null (completed downloads move to history). Transition detection is now a standalone check at the top of `render()` that tracks the previously-current download by ID.
- **Silero VAD first-run download had no user feedback** — the ~10 MB torch.hub download ran silently with raw log lines. A new `loading-vad` stage event emits a localized message so users know what is being downloaded and why.

## Changed

- **Download guide cleaned up** — removed bilingual text and emoji from `.github/release-download-guide.md` so release pages are cleaner.

No user action required.
