# v1.15.9

Recent-transcripts auto-prune (parity with the v1.15.5 recent-folders dropdown) and two console-log classifier fixes.

## Added

- **Recent-transcripts auto-prune** ([src/main/ipc-handlers.js](src/main/ipc-handlers.js), [preload/preload.js](preload/preload.js), [src/renderer/components/history.js](src/renderer/components/history.js), [src/renderer/index.html](src/renderer/index.html), [locales/*/progress.json](locales/zh-TW/progress.json)) — history rows whose source media **and** transcript output are both gone are now removed at three moments: app startup, after each new transcription is added, and on a new per-tap **Refresh** button in the history header. Conservative policy: if either the source or the subtitle is still on disk the row stays, so merely deleting `.srt` doesn't lose the "Show in folder" affordance. Mirrors `fs:filter-existing-dirs` / recent-directories pruning introduced in v1.15.5.

## Fixed

- **Console classifier no longer red-flags Whisper transcript lines containing "error"** ([src/renderer/components/console-log.js](src/renderer/components/console-log.js)) — lines matching the SRT timestamp pattern `[MM:SS.xxx -> MM:SS.xxx]` are now short-circuited to the neutral colour before any keyword check. Previously, if the speaker actually said "error" (or "失敗") in the audio, the transcribed subtitle line would be classified as a system error and turn red.

- **`✔ 批次完成` success summary is no longer classified as an error** ([src/renderer/components/console-log.js](src/renderer/components/console-log.js)) — check-mark-prefixed lines (`✓` / `✔`) are now recognized as success summaries *before* the error-keyword pass, so a summary like "✔ 批次完成：1 完成、0 失敗、0 略過" is correctly green even though the line legitimately contains "失敗" with a zero count.

- **Traditional Chinese typography — middle dot spacing** ([locales/zh-TW/events.json](locales/zh-TW/events.json)) — `Whisper 轉錄中·模型 {{model}}` → `Whisper 轉錄中 · 模型 {{model}}`. Matches the English spacing convention.
