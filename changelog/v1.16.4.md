# v1.16.4

Fixes whisper subtitle lines being mis-coloured red / green in the console log when the spoken content happened to contain "錯誤" / "完成".

修正 console log 中 whisper 字幕行因口語內容含「錯誤」「完成」而被誤判成紅 / 綠色的問題。

## Fixed / 修正

- **Subtitle lines mis-coloured by spoken keywords** ([src/renderer/components/console-log.js](src/renderer/components/console-log.js)) — `classifyLine()` short-circuits whisper subtitle rows to neutral so that words like "錯誤" / "完成" inside the transcribed speech don't trigger the error / ok colour. The short-circuit regex only recognised two-segment timestamps (`[MM:SS.mmm -> MM:SS.mmm]`) and failed against the actual three-segment format (`[HH:MM:SS.mmm -> HH:MM:SS.mmm]`), so subtitle rows fell through to keyword classification and were coloured red or green based on whatever the speaker happened to say. The pattern now accepts both two- and three-segment timestamps.
  **字幕行因口語關鍵字被誤上色** — `classifyLine()` 對 whisper 字幕行有「強制中性色」的短路邏輯，避免轉錄內容裡的「錯誤」「完成」觸發紅 / 綠色。但短路用的 regex 只認兩段式時間戳 (`[MM:SS.mmm -> MM:SS.mmm]`)，遇到實際的三段式格式 (`[HH:MM:SS.mmm -> HH:MM:SS.mmm]`) 直接 fail，使字幕行落入後續關鍵字判斷，講話內容含「錯誤」就變紅、含「完成」就變綠。改為兩段、三段時間戳皆接受。
