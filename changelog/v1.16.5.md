# v1.16.5

Replaces the wall of ffmpeg stderr shown when a file is moved/renamed/deleted mid-transcription with a clear bilingual "input file no longer exists" banner.

當音訊檔案在轉錄期間被移動、改名或刪除時，不再顯示一大坨 ffmpeg stderr，改顯示乾淨的雙語錯誤訊息。

## Improved / 改善

- **Detect "input file vanished mid-transcription"** ([python/whisperflow/vad/base.py](python/whisperflow/vad/base.py), [python/whisperflow/cli.py](python/whisperflow/cli.py), [python/whisperflow/events.py](python/whisperflow/events.py), [src/main/ipc-handlers.js](src/main/ipc-handlers.js), [src/main/error-catalog.js](src/main/error-catalog.js), [locales/en/errors.json](locales/en/errors.json), [locales/zh-TW/errors.json](locales/zh-TW/errors.json)) — `transcriber.py` only checks `is_file()` once at job start, but VAD and Whisper segments respawn ffmpeg every chunk; if the user renames or moves the source folder mid-run, the chunk-level ffmpeg invocation fails with a multi-line stderr that buries the actual cause. `load_audio()` and `get_audio_duration()` now do a pre-check + post-check around each ffmpeg/ffprobe call: if the file is missing on entry — or if ffmpeg fails AND the file is no longer at the original path (race condition) — they raise a new `InputFileVanishedError` carrying the path. The CLI catches it specifically and emits a structured event with `reason: input_file_vanished` plus the i18n key `errors:INPUT_FILE_VANISHED.message`. The main process maps that to a new `INPUT_FILE_VANISHED` error code, and the renderer shows "Input file no longer exists / 輸入檔案已不存在 — The audio file was moved, renamed, or deleted during transcription: {path}".
  **轉錄途中檔案消失時改顯示乾淨訊息** — `transcriber.py` 只在 job 開始時用 `is_file()` 檢查一次，但 VAD 與 Whisper 每處理一個 chunk 都會重新 spawn ffmpeg；使用者中途把來源資料夾改名或搬走時，chunk 層級的 ffmpeg 呼叫會失敗、吐出一大坨 stderr 把真正原因蓋掉。`load_audio()` 與 `get_audio_duration()` 現在對每次 ffmpeg/ffprobe 呼叫做 pre-check + post-check：進入前若檔案不在、或 ffmpeg 失敗且原路徑檔案已不存在（race condition），就拋新的 `InputFileVanishedError` 並帶上路徑。CLI 專屬捕獲這個 exception，發出帶 `reason: input_file_vanished` 與 i18n key `errors:INPUT_FILE_VANISHED.message` 的結構化事件。Main 進程映射為新的 `INPUT_FILE_VANISHED` 錯誤碼，UI 顯示「輸入檔案已不存在 — 音訊檔案在轉錄期間被移動、改名或刪除：{path}」。
