# v1.8.2

Patch release that makes the **model-not-downloaded guard truly mandatory** by moving it into the main-process `run:cli` handler so it covers every code path — button click, auto-loop after scan, auto-loop next file, skip-and-continue.

## Fixed

- **Model-missing guard was still bypassable via auto-loop** — v1.8.1 fixed the guard reading from the DOM (instead of `config.json`), but the guard only lived in the renderer's `ensureRunReady()` which is called on the initial "Run Transcription" button click. Three other code paths in `controls-bar.js` call `window.electronAPI.runCli()` directly for auto-loop / queue-retry / skip-and-continue, and those never went through `ensureRunReady()`. The result: if auto-loop was enabled, the second transcription attempt would bypass the guard and let `faster-whisper` silently download the model from HuggingFace.

  The fix moves the model check into the **main-process `run:cli` IPC handler** (`ipc-handlers.js`), right after the venv check and before `startNextJob()`. This is the single choke point that every `runCli()` call goes through, so no renderer code path can bypass it. The check reads `config.json :: SETTING.model` and walks the managed models directory for the required files (`config.json`, `tokenizer.json`, `vocabulary.txt`, `model.bin` or `model.safetensors`) — a pure filesystem check, no Python spawn needed.

  When the model is missing, the handler:
  1. Broadcasts `run:model-missing` to the renderer with the model name
  2. Logs a clear message to the console
  3. Sends `run:done` with code 1 so the auto-loop stops

  The renderer listens for `run:model-missing` and shows the same themed confirm dialog ("模型 large-v2 尚未下載…") that the old renderer-side guard would have shown, offering to switch to the Models tab. This dialog is wired through `controls-bar.js` so it fires regardless of which code path triggered the run.

- **Guard no longer silently falls through on errors** — the old renderer-side `try/catch` swallowed any exception from `listModels()` and let the transcription proceed. The new main-side check uses a `try/catch` that **blocks the run** and shows a `MODEL_CHECK_FAILED` error dialog instead of silently falling through.

## Migration notes

No user action required. The renderer-side guard in `ensureRunReady()` is still there as a first line of defence for the button-click path, but the main-process gate is now the authoritative check.
