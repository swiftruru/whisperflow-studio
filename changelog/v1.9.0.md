# v1.9.0

Feature release focused on post-transcription UX and workflow efficiency.

## Added

- **Batch completion summary** — when a batch finishes (or auto-loop exhausts the queue), a green summary line is written to the Console showing done / failed / skipped counts and total elapsed time. The Batch Progress card headline also updates with the same statistics instead of the generic "All queued files processed" text.
- **Enhanced system notifications** — the OS notification sent when auto-loop completes now includes batch statistics (done, failed, elapsed) instead of a generic "All files transcribed!" message.
- **Output file quick access** — completed queue items now show a "Show in folder" button that reveals the source media file (and its generated subtitles) in the system file manager. History rows also gain a 📂 button for the same action.
- **Keyboard shortcuts for core actions** — `⌘R` / `Ctrl+R` to run transcription, `⌘⇧S` / `Ctrl+Shift+S` to scan for missing subtitles, `⌘.` / `Ctrl+.` to stop the batch. All three are listed in the `?` shortcuts modal.
- **Drag-and-drop individual files** — dropping media files (not a folder) onto the directory card now adds them directly to the transcription queue instead of scanning the parent directory. Supported formats are validated, duplicates and files with existing subtitles are skipped, and a toast reports how many files were added.
- **Colored transcription completion log** — the Python backend now prints a bold green `✔ Transcription completed in X.XXs` line to the console after each file finishes, making it much easier to spot completion in a long log.

## Fixed

- **Onboarding spotlight dimmed the highlighted element** — both the backdrop overlay and the spotlight's `box-shadow` were darkening the screen simultaneously, causing the highlighted element to appear dimmed through the backdrop. The backdrop is now hidden when a spotlight target is active, so only the `box-shadow` provides the darkening effect and the highlighted element stays bright.

## Migration notes

No user action required.
